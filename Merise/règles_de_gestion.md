<h1 align="center">Les règles de gestion</h1>

**RG1 :** Une commune doit avoir un code INSEE, un nom et un libellé d’acheminement.  
**RG2 :** Positions doit avoir un type de position.  
**RG3 :** Une ancienne commune doit avoir un nom.  
**RG4 :** Une Source_position doit avoir un libellé.  
**RG5 :** Une adresse doit avoir un numéro, un nom de voie, un code postal, un code INSEE d’ancienne commune, un x, un y, une longitude, une latitude, un nom afnor, une valeur de 1 ou 0 de certification, un nom de ld et un cad parcelle.  
**RG6 :** Positions peut être associé de 1 à n adresse.  
**RG7 :** Une adresse est associée à 0 ou 1 seule position.  
**RG8 :** Une source_position peut définir de 0 à n adresses.  
**RG9 :** Une adresse peut être définie par 0 à 1 source_position.  
**RG10 :** Une ancienne commune comporte de 0 à n adresse.  
**RG11 :** Une adresse est comportée dans une et une seule ancienne commune.  
**RG12 :** Une adresse est rattachée à une et une seule commune.  
**RG13 :** une commune peut rattacher à 1 ou n adresses.  
**RG14 :** Une adresse est liée un et un seul rep.  
**RG15 :** Un rep est lié à 0 à n adresse.  
**RG16 :** Un cad est attaché à une et une seule adresse.  
**RG17 :** Une adresse attache 0 à n cad.
